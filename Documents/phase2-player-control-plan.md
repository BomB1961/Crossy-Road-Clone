# 2단계: 플레이어 조종 구현 계획 (모바일 3D)

## 모바일 고려사항

| 항목 | 적용 내용 |
|------|----------|
| **입력** | 전체 화면 터치 (Tap/Swipe) |
| **좌표** | 쿼터뷰 3D 타일 기반 |
| **성능** | 오브젝트 풀링, 불필요 Update 최소화 |

---

## 1. 입력 감지 메커니즘

### 터치 판정 (On Finger Up 기준)

| 조건 | 동작 |
|------|------|
| 거리 < 50px | **Tap** → 1칸 전진 |
| 거리 ≥ 50px, 상하 | **Swipe Up/Down** → 전진/후진 |
| 거리 ≥ 50px, 좌우 | **Swipe Left/Right** → 측면 이동 |

### 가속 기능 제외
- 롱 프레스/연속 가속 없음
- 연타를 통한 이동 권장

---

## 2. 3D 좌표 및 방향 매핑

### 월드 방향 벡터

| 화면 입력 | 월드 방향 | Vector3 |
|----------|-----------|---------|
| Up / Tap | Forward (+Z) | (0, 0, 1) |
| Down | Back (-Z) | (0, 0, -1) |
| Right | Left (+X) | (1, 0, 0) |
| Left | Right (-X) | (-1, 0, 0) |

### 캐릭터 회전
- 이동 방향을 즉시 바라본 뒤 점프 애니메이션 실행

---

## 3. 이동 및 물리 규칙

### 그리드 스냅
- 이동 후 좌표 = **정수값** (예: 1.02 → 1.0)

### 입력 큐 (Input Queueing)
- 점프 중 입력 저장 (최대 **1개**)
- 착지 후 즉시 다음 점프 실행

### 장애물 체크 (Raycast)
- 점프 전 목적지 레이캐스트
- 장애물(나무, 바위) 있으면 이동 차단, 제자리 점프만 실행

---

## 4. 카메라 제어

| 규칙 | 설명 |
|------|------|
| **추적** | X/Z축 따라 부드럽게 이동 |
| **후진 미추적** | Down 입력 시 카메라 고정 |
| **데드존** | 화면 하단 경계 밖 → 독수리 등장 → 게임 오버 |

---

## 구현 순서

1. **PlayerInputHandler** - 터치 감지, Tap/Swipe 판정
2. **PlayerController** - 이동, 그리드 스냅, 입력 큐, 회전
3. **TerrainGenerator** - 레인 생성/관리
4. **GrassLane** - 풀밭 레인
5. **RoadLane** - 도로 레인
6. **Obstacle** - 레이캐스트 장애물 체크

---

## 스크립트 책임 분장

| 스크립트 | 책임 |
|---------|------|
| PlayerInputHandler | 터치 입력 감지 → 방향 enum 전달 |
| PlayerController | 이동 실행, 좌표 스냅, 입력 큐, 회전 |
| TerrainGenerator | 레인 타입별 생성, 장애물 배치 |
| BaseLane | 레인 기본 관리 |
| BaseObstacle | 장애물 기본 관리 |
