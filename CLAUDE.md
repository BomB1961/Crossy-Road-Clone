# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Type

Unity 3D game project (version 6000.4.0f1, 2024 LTS) — a Crossy Road-style endless runner clone. **No game code has been implemented yet**; the project is a starter shell with infrastructure configured.

## Build & Development

- **Open in Unity**: Open `Crossy Road Clone.sln` in Visual Studio or launch Unity and open this directory as a project.
- **Build**: Use Unity's Build Settings (File > Build Settings). No custom build pipeline exists.
- **No tests**: The project currently has no test suite configured.
- **Input actions**: `Assets/InputSystem_Actions.inputactions` defines Player and UI action maps. Do not edit this file manually; use Unity's Input Action Asset editor.

## Architecture

### Current State
The project has infrastructure configured but zero custom scripts. All C# logic needs to be created from scratch.

### Configured Systems
- **URP (Universal Render Pipeline)** v17.4.0 with dual profiles:
  - `Settings/PC_RPAsset` / `PC_Renderer` — desktop quality
  - `Settings/Mobile_RPAsset` / `Mobile_Renderer` — mobile optimization (0.8x render scale)
- **Post-processing**: Global Volume in SampleScene uses `Settings/SampleSceneProfile.asset`
- **Unity Input System** v1.19.0 with pre-defined actions (Move, Jump, Sprint, etc.) across Keyboard&Mouse, Gamepad, Touch, Joystick, XR bindings

### Package Dependencies
- `com.unity.ai.navigation` (2.0.11) — Navigation mesh
- `com.unity.visualscripting` (1.9.10) — Visual scripting (Bolt successor)
- `com.unity.timeline` (1.8.11) — Animation/cutscene timeline
- `com.unity.multiplayer.center` (1.0.1) — Multiplayer (placeholder for future)
- `com.unity.mathematics` — Mathematics utility

### Scene
- `Assets/Scenes/SampleScene.unity` — Default scene containing Main Camera, Directional Light, and Global Volume

## Expected Game Structure (not yet implemented)

To build the Crossy Road clone, these core systems are needed:
1. **Player Controller** — tile-based or continuous movement handling jumps/sprints
2. **Road/Terrain Generator** — procedural infinite terrain or pooled tile system
3. **Obstacle Spawner** — cars, trucks, trains with collision detection
4. **Score System** — distance-based scoring
5. **Game Over / Restart Logic** — death on collision, retry flow
6. **UI** — HUD for score display

## Key Paths
- `Assets/Scenes/SampleScene.unity` — main scene
- `Assets/InputSystem_Actions.inputactions` — input action definitions
- `Assets/Settings/` — URP and post-processing configuration
